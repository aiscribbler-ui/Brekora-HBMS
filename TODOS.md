## Design System Rollout

### [ ] Validate curated palette for WCAG AA contrast
- **Why:** Success criterion requires zero contrast violations on guest-facing pages. Colors picked by eye may fail WCAG AA against white or dark backgrounds.
- **What:** Run a contrast checker script (e.g., `wcag-contrast` npm package) against `#ffffff` and `#111827` for all 20-30 curated palette candidates. Reject any color below 4.5:1 for normal text or 3:1 for large text.
- **Depends on:** Phase B (palette selection).
- **Blocked by:** None.

### [ ] Implement theme store with localStorage fallback
- **Why:** No theme store currently exists despite `darkMode: 'class'` in Tailwind config. Dark mode classes exist in some components but the `dark` class is never applied to `document.documentElement`. Safari/Firefox private mode also blocks localStorage access.
- **What:** Create a Zustand theme store that detects `prefers-color-scheme`, toggles the `dark` class on `document.documentElement`, and persists to `localStorage` with a try/catch fallback for private mode.
- **Depends on:** Phase C1 (theme store implementation).
- **Blocked by:** None.
